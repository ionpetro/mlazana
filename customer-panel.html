In Prosperty we call it Customer panel. It is a product that helps a buyer or a property owner to be in charge of the selling process. Users can manage scheduled viewings, offers, and mortgages, gather favorite properties, and save searches.

ROLE: TEAM LEADER

I led the Product Design team. I reported directly to the CEO and collaborated closely with the CTO, Marketing manager and the Development team.

ABOUT PROSPERTY

Real estate is one of the few industries that has failed adapt to technological changes in the last many years, both in terms of cost-value and customer experience. Time-consuming and complicated procedures, as well as lack of data and transparency, make any transaction displeasing for every party involved.
Our solution? A reliable digital platform, that brings trust and convenience in every real estate transaction, offering data accuracy, transparency, effectiveness and credibility.

THE CHALLENGE

The biggest challenge was to make a scalable product for entirely different personas that could be separated or combined. We have a buyer (one persona with many profiles and intentions) and a property owner, which sometimes can be the same person. We managed to create a general panel for every persona but, simultaneously, smart and personalized. We applied the pages needed in a sidebar, and we kept one important rule:
- If a user has a listed property, then he/she is definitely an owner (and a potential buyer); otherwise, the user is, at the moment, a buyer.
With that rule, we succeeded in having a panel that shows and hides components based on the persona. For example, inside the ‚ÄúMy offers‚Äù page, the user can see the components that match their intentions.

Case 1: Property owner

If the user has listed property, then the platform recognizes it as a property owner.

Case 2: Buyer

If the user place an offer then the platform consider him as a buyer.

Case 3: Property owner and buyer

If the user has listed property and also places offers in other properties.

DASHBOARD

Dashboard has three main goals: 1. Showcase an overview of your schedules appointments and properties 2. Inform user about the urgent tasks that need to be done 3. Promote marketing material about Prosperty's services

FAVOURITE PROPERTIES

Having a list of all your favorite properties is beneficial while searching for the ideal one. So we created a page with the properties added to favorites through the search page.

SAVE SEARCH

Saved searches refer to search queries a user makes. These queries are beneficial as we used them for creating personalized marketing content with weekly property suggestions that match.

SCHEDULE A TOUR

A buyer can schedule a tour online to view a property. If the agent accepts the tour, then the owner gets notified. Both personas have all the tours collected in the customer panel.

MAKE AN OFFER

A buyer can make an offer on a property. All offers are collected in the customer panel for both personas, buyers, and property owners. In the next versions of the platform, the owner can accept, decline, or make a counteroffer to the buyer.

MAKE AN OFFER

Lastly, the customer panel has all the submitted listings of a property owner. For every listing, the owner can:
  • View analytics about leads, total viewings, etc.
  • View scheduled tours per property
  • View offers per property
  • View the closing process if an offer is accepted)
